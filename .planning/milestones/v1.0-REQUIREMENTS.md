## v1 Requirements

All 45 requirements shipped in v1.0 MVP.

### Authentication

- [x] **AUTH-01**: Admin and pegawai can login with username and password — ✅ Delivered
- [x] **AUTH-02**: Admin can upload Excel file with employee data (NIP, Nama, Unit) — ✅ Delivered
- [x] **AUTH-03**: System auto-generates employee accounts (username=NIP, password=NIP) — ✅ Delivered
- [x] **AUTH-04**: System shows password change reminder when pegawai uses default NIP password — ✅ Delivered
- [x] **AUTH-05**: Pegawai can change their own password — ✅ Delivered
- [x] **AUTH-06**: Admin can download Excel template for employee import — ✅ Delivered

### Stock Management

- [x] **STOCK-01**: Admin can manually add new items with category, name, initial quantity — ✅ Delivered
- [x] **STOCK-02**: Admin can manually edit item details and adjust quantities — ✅ Delivered
- [x] **STOCK-03**: Admin can upload Excel file to bulk restock items — ✅ Delivered
- [x] **STOCK-04**: System validates Excel format and quantity values before import — ✅ Delivered
- [x] **STOCK-05**: System shows preview of Excel data before final save — ✅ Delivered
- [x] **STOCK-06**: Admin can download Excel template for restock import — ✅ Delivered
- [x] **STOCK-07**: Admin can categorize items by type (writing tools, paper, etc.) — ✅ Delivered
- [x] **STOCK-08**: System tracks three stock states: Available, Reserved, Used — ✅ Delivered
- [x] **STOCK-09**: System shows low stock alert when available stock below threshold — ✅ Delivered
- [x] **STOCK-10**: System blocks new requests when available stock is zero — ✅ Delivered
- [x] **STOCK-11**: Admin can view current stock levels with breakdown (available/reserved/used) — ✅ Delivered

### Request Management

- [x] **REQ-01**: Pegawai can create request by selecting items and quantities — ✅ Delivered
- [x] **REQ-02**: System validates requested quantity against available stock — ✅ Delivered
- [x] **REQ-03**: Pegawai can view list of own requests with status — ✅ Delivered
- [x] **REQ-04**: Pegawai can view details of each request (items, quantities, status, notes) — ✅ Delivered
- [x] **REQ-05**: Admin can view list of all requests with filters (pending/approved/delivered/rejected) — ✅ Delivered
- [x] **REQ-06**: Admin can approve request and reserve stock — ✅ Delivered
- [x] **REQ-07**: Admin can modify requested quantity when approving and add reason note — ✅ Delivered
- [x] **REQ-08**: Admin can reject request with reason note — ✅ Delivered
- [x] **REQ-09**: System reserves approved quantities (deduct from Available, add to Reserved) — ✅ Delivered
- [x] **REQ-10**: Admin can mark items as delivered using checklist interface — ✅ Delivered
- [x] **REQ-11**: System deducts delivered quantities from Reserved and adds to Used — ✅ Delivered
- [x] **REQ-12**: System auto-cancels undelivered quantities (return to Available from Reserved) — ✅ Delivered
- [x] **REQ-13**: Request status tracked through lifecycle: pending → approved → delivered/rejected — ✅ Delivered
- [x] **REQ-14**: Pegawai can cancel own pending request (before admin approval) — ✅ Delivered

### Notifications

- [x] **NOTIF-02**: Pegawai receives in-app notification when request is rejected — ✅ Delivered

- [x] **NOTIF-04**: Admin receives in-app notification when new request is submitted — ✅ Delivered
